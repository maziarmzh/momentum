Detailed Description

template<typename T>
class momentum::OnlineBlockHouseholderQR< T >

Online Householder QR solver for least-squares problems with block-arrowhead structure.

Solves

min || A * x - b ||_2
momentum::Vector3
Vector< T, 3 > Vector3
Definition types.h:235

where A has the block structure

[ A_11 A_1n ]
[ A_22 A_2n ]
[ A_33 A_3n ]
[ ... ... ]

i.e. independent diagonal blocks A_ii (one set of disjoint x_i per block) plus a shared set of "common" columns A_in that are used by every block.

This is the case for calibration problems, where each hand pose is independent but a shared set of calibration parameters connects them.

Besides the blockwise structure, the behavior of this solver is identical to OnlineHouseholderQR — see that class's notes for the algorithmic details. The key takeaway is that this can be used in an online fashion: poses are passed in one at a time and the Jacobians can then be thrown away.

Note
A future version could also discard the intermediate R_ii_ and R_in_ blocks if memory becomes an issue (only R_nn_ is required to compute x_n()).

Usage contract

  1. Construct with the number of common unknowns n_common and (optionally) lambda damping. Diagonal block sizes are inferred from each add() call.
  2. Call add()/addMutating() once per block, supplying a unique iBlock index. Internal R_ii_, R_in_, R_nn_, y_i_, and y_n_ are mutated incrementally.
  3. Retrieve results via x_dense(), x_i(iBlock), or x_n().
  4. Call reset() to discard accumulated state and reuse the solver.

Constructs a solver for problems with n_common shared unknowns.

Per-block unknown counts are determined from the first add() call for each block. lambda implements LM-style damping, emulating

| [ lambda * I ] [ x ] - [ 0 ] |^2
| [ A ] [ b ] |

without materializing the extra rows.
